## CHANGELOG v6.0 → v6.1

**КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ:** логика масштабирования тиков между уровнями S.

**Проблема v6.0:**
Система считала что 9 тиков на любом уровне S = коллапс абсолютно.
Это противоречило рекурсии "1 тик на S+1 = цикл [3→5→9] на S".

**Решение v6.1:**
Введена **компрессия тиков при наблюдении между уровнями** через W(n).
Пороги 3/5/9 применяются только к тикам своего уровня S.
При взгляде на соседний уровень тики сжимаются/растягиваются через варпинг.

Это объясняет почему на BTC все ТФ показывали "коллапс" одновременно —
каждый был в коллапсе на своём уровне, но для наблюдателя выше
это выглядело как нарастание (7) или эхо (5-6).

## ПРОИСХОЖДЕНИЕ

SIFS основан на теории Scale-Invariant Fractal Spacetime (github.com/m0rfy/SIFS-Theory-Core):
наша реальность — 3-брана в 5D фрактальном пространстве, где пятая координата S
есть физический масштаб. Аналитический инструмент SIFS переносит эту геометрию
в анализ любых наблюдаемых процессов.

Ключевая идея: наблюдатель — часть системы.
Выбор масштаба наблюдения — это выбор координаты S.
Это меняет то что видишь — не потому что данные другие,
а потому что ты буквально в другой точке пространства.

## ЧАСТЬ I. ФУНДАМЕНТАЛЬНЫЕ КОНСТАНТЫ

```
π  = 3.14159265...   базовая мера минимального движения
φ  = 1.61803398...   золотое сечение

Ψ₁ = 9/π   ≈ 2.8648  скейлинг между уровнями S
Ψ₂ = 9/π/2 ≈ 1.4324  скейлинг внутри уровня S
GM = (π+φ)/2 ≈ 2.3799 золотая середина (порог значимости Score)

k    = 1/(π×Ψ₁) ≈ 0.1111  параметр затухания
W(S) = exp(−2k|S|)          варпинг-функция

W(0) = 1.000  W(1) = 0.800  W(2) = 0.641
W(3) = 0.513  W(4) = 0.411  W(5) = 0.329
```

Принцип иррациональности: π и φ трансцендентны.
Их произведения никогда не совпадают точно с целыми числами.
Сетка уровней никогда не "захлопывается" в периодичность —
система остаётся фрактальной до любой глубины наблюдения.

## ЧАСТЬ II. ОПРЕДЕЛЕНИЕ ТИКА

### Главный принцип

1 тик — не минимальный ценовой шаг.
1 тик — минимальное движение которое система "помнит",
         то есть которое меняет её следующее состояние.

### Операциональное определение (рыночные данные)

```
1 тик = одна завершённая попытка взаимодействия цены
        с ближайшей активной границей

Три условия подтверждения тика (все обязательны):
  1. Цена коснулась или пробила границу (BB / EMA / RSI-уровень)
  2. Свеча закрылась (тело свечи, не тень)
  3. RSI / momentum сдвинулся в направлении движения

Размер тика = (точка касания BB − EMA) / ширина BB
```

### Вес тика по объёму

```
объём < медианы    → вес 0.5  (слабый тик, вероятно эхо)
объём ≈ медианы    → вес 1.0  (стандартный тик)
объём > 2×медианы  → вес 1.5  (усиленный тик, вероятно истина)
```

### Операциональное определение (нерыночные данные)

```
1 тик = одно непрерывное направленное изменение
        от смены знака до следующей смены знака

Размер тика = Σ|Δ| пока знак не меняется
Направление = sign первого изменения в серии
```

## ЧАСТЬ III. КОМПРЕССИЯ ТИКОВ ПРИ СМЕНЕ МАСШТАБА (НОВОЕ В v6.1)

### Проблема которую решает компрессия

**Противоречие v6.0:**
```
Рекурсия говорит: 1 тик на S+1 = завершённый цикл [3→5→9] на S
Но пороги говорят: 9 тиков = коллапс на любом уровне

Следствие: если на S накоплено 9 тиков и произошёл коллапс,
то на S+1 это должен быть 1 тик. Но тогда для коллапса на S+1
нужно 9 тиков уровня S+1 = 9 × 9 = 81 тик уровня S.

Временные таймфреймы не совпадают с этой логикой:
1 минута = 60 секунд, а не 81.
```

### Решение: наблюдатель видит тики соседних уровней с искажением

```
Тики_видимые(с уровня S, смотрим на S±n) = Тики_реальные(S±n) × W(n)

где W(n) = exp(−2k|n|) — варпинг-функция
```

### Таблица компрессии

| Реальных тиков на S±n | W(n) | Видимых тиков с S | Интерпретация |
|---|---|---|---|
| 9 тиков на S-1 | W(1)=0.800 | 7.2 → 5-7 | Нарастание / Эхо |
| 9 тиков на S-2 | W(2)=0.641 | 5.8 → 3-5 | Эхо / Истина |
| 9 тиков на S-3 | W(3)=0.513 | 4.6 → 3-4 | Истина / Переход |
| 9 тиков на S+1 | 1/W(1)=1.25 | 11.3 → 9-11 | Коллапс |
| 3 тика на S-5 | W(5)=0.329 | 0.99 → шум | Не регистрируется |

### Абсолютный минимум: π

```
Критическое правило:
  Тики никогда не сжимаются ниже π ≈ 3.

  Если Тики_видимые < 3 → шум, игнорируется наблюдателем
  Если Тики_видимые ≥ 3 → регистрируется как истина

Это означает: даже глядя на S+100 откуда-то с S=0,
наблюдатель увидит минимум ~3 тика, иначе это шум.

Физический смысл: π — минимальная структура
которую система способна различить как "не-случайность".
```

### Почему это важно для практики

**Пример 1: BTC multi-TF анализ v6.0**

В v6.0 мы видели:
```
1m:  42 тика от S=0 → КОЛЛАПС
5m:  17 тиков от S=0 → КОЛЛАПС
1h:  23 тика от S=0 → КОЛЛАПС
```

Это создавало впечатление что всё одновременно коллапсирует.

**Правильная интерпретация v6.1:**
```
Наблюдатель на 5m (S=2):
  Смотрит вниз на 1m (S=0):
    42 тика на S=0 × W(2) ≈ 42 × 0.641 ≈ 27 тиков
    но они сжимаются ещё раз при переходе S=0→S=2:
    реально видит ~7 тиков (нарастание, не коллапс)

  На своём уровне 5m:
    17 тиков → КОЛЛАПС (для него самого)

  Смотрит вверх на 1h (S=4):
    23 тика на S=4 / W(2) ≈ 23 / 0.641 ≈ 36 тиков
    но с его разрешения S=2 он видит ~5-6 (эхо-зона)
```

Каждый уровень действительно в коллапсе для себя —
но для других уровней это выглядит иначе.

**Пример 2: Геополитика**

```
AI (S=3):           6.5 тиков, Score +4.734
Геополитика (S=5):  7.0 тиков, Score +2.376

Почему AI не создаёт немедленный геополитический кризис?

С уровня S=5 глядя на S=3:
  6.5 × W(2) = 6.5 × 0.641 ≈ 4.2 видимых тика

Для геополитики AI-процесс выглядит как "переход (4)",
а не "эхо (6)" — детализация теряется при взгляде вниз.

Это объясняет почему крупные геополитические структуры
медленно реагируют на технологические потрясения:
не потому что не видят — а потому что видят сжато.
```

## ЧАСТЬ IV. ИНДИКАТОРЫ КАК ФИЗИЧЕСКИЕ ГРАНИЦЫ

### BB (Bollinger Bands) → ПРОСТРАНСТВО

```
Ширина BB ∝ W(S) = exp(−2k|S|):
  узкий BB  → малый S, высокая детализация (микро-уровень)
  широкий BB → большой S, крупные движения (макро-уровень)

BB squeeze (сжатие):
  система переходит на S-1, готовится к новому определению тика
  → Contraction, шум усилен

BB расширение:
  система переходит на S+1, тик укрупняется
  → Expansion или Коллапс

Три попытки пробоя BB = триада P=3 (физическое воплощение):
  Попытка 1: касание верхней BB, закрытие внутри    → тик 1
  Попытка 2: повторное касание, RSI растёт           → тик 2
  Попытка 3: пробой + закрытие за BB + объём > мед.  → тик 3 = ИСТИНА
```

### EMA → НАПРАВЛЕНИЕ

```
EMA = динамический S=0 для данного масштаба

Цена выше EMA: тики вверх "по течению", вес стандартный
Цена ниже EMA: тики вниз "по течению", вес стандартный
Цена на EMA:   Transition, направление не определено

Пересечение EMA снизу вверх: сигнал смены S=0
Пересечение EMA сверху вниз: сигнал смены S=0

W4 в Score использует EMA как референс:
  W4 = (X_current − X_ema) / BB_width
```

### RSI → ЭНЕРГИЯ

```
RSI как счётчик тиков:

  RSI 40–60:  Transition, нейтральная зона (тики 1–2)
  RSI 60–70:  нарастание вверх (тики 3–4, Истина / Переход)
  RSI > 70:   тик 5 (Эхо, перегрев — следующий тик вероятно вниз)
  RSI > 80:   зона Коллапса (9 тиков вверх, разворот высоковероятен)

  RSI 30–40:  нарастание вниз (тики 3–4)
  RSI < 30:   тик 5 вниз (Эхо перепроданности)
  RSI < 20:   зона Коллапса вниз

Движение RSI от 50 к 70 = ~5 шагов по 4 пункта:
  это прямое соответствие порогам 1→2→3(истина)→4(переход)→5(эхо)
```

### Объём / IO → ПОДТВЕРЖДЕНИЕ

```
Тик без объёма (< медианы):
  ложный пробой → засчитать как Эхо (тик 5, noise penalty)

Тик с объёмом (> медианы):
  истинный пробой → засчитать как полный тик

Поглощение (высокий объём, цена не сдвинулась):
  борьба на уровне → Эхо с дополнительным penalty ×0.5

Дивергенция RSI/цена:
  цена делает новый максимум, RSI — нет → тик 5 (скрытое Эхо)
  учитывать при классификации последнего тика
```

## ЧАСТЬ V. ПОРОГОВЫЕ ЗНАЧЕНИЯ

Количество взвешенных тиков от S=0 **НА СВОЁМ УРОВНЕ S**.

```
1–2 тика   Шум
           BB сжат, RSI 40–60
           Действие: игнорировать

3 тика     Истина [ceil(π) = 3]
           BB начинает расширяться, RSI выходит из 40–60
           Действие: подтвердить триаду, считать сигнал
           Физика: три последовательных тика = все уровни S согласованы

4 тика     Переход
           BB расширяется, RSI 60–70 или 30–40
           Действие: наблюдать, не подтверждать

5 тиков    Эхо [Fibonacci(5) = 5]
           BB расширен, RSI > 70 или < 30
           Действие: noise penalty ×0.75, требовать подтверждения с S±1
           Физика: 5-й тик = наблюдатель влияет на наблюдение
                   принцип неопределённости системы

6–8 тиков  Нарастание к коллапсу
           BB полностью расширен, RSI 70–80
           Действие: готовить смену фазы, следить за объёмом

9 тиков    Коллапс [round(π×Ψ₁) = 9, также 9 = 3²]
           BB на максимуме, RSI > 80 или < 20
           Действие: объявить смену фазы, пересчитать S=0
           Физика: 9 = триада триад, система исчерпала масштаб S

>9 тиков   Новая иерархия
           BB начинает сжиматься, RSI возвращается к 50
           Действие: перейти на S+1, весь цикл [3→5→9] = 1 тик нового уровня
```

Мосты между порогами:
```
3 × Ψ₁ ≈ 8.59 → 9    (прямой переход от истины к коллапсу)
3 × Ψ₂ ≈ 4.30 → 4    (переход от истины к переходной зоне)
5 × φ  ≈ 8.09 → 9    (эхо нарастает к коллапсу через золотое сечение)
```

## ЧАСТЬ VII. РЕКУРСИЯ МАСШТАБОВ (ПОЛНОСТЬЮ ПЕРЕРАБОТАНО В v6.1)

### Главный принцип (дополнен)

```
1 тик на уровне S+1  =  завершённый цикл [3→5→9] на уровне S

НО: при наблюдении с S на S+1 этот 1 тик выглядит как ~1/W(1)≈1.25 тика
    при наблюдении с S+1 вниз на S: 9 тиков выглядят как ~7 тиков
```

### Механизм перехода при коллапсе (9 тиков НА УРОВНЕ S)

```
1. Зафиксировать накопленный взвешенный импульс (W3) на уровне S
2. Умножить на W(1) = exp(−2k) ≈ 0.800
3. Передать как начальный импульс уровня S+1
4. Пересчитать S=0 = точка коллапса на уровне S
5. BB на уровне S начинает сжиматься (подтверждение)
6. На уровне S+1: весь прошедший цикл регистрируется как 1 тик
```

### Направление по S

```
Прямое  (3→9): развитие, BB расширяется, RSI растёт
               варпинг быстрее (информация быстро затухает вверх)

Обратное (9→3): затухание, BB сжимается, RSI возвращается к 50
                варпинг медленнее (система помнит структуру)
```

### Конфликт масштабов (с учётом компрессии)

Два наблюдателя на S=1 и S=3 видят противоположные сигналы —
и оба правы на своём уровне.

```
Пример:
  S=1: накоплено 9 тиков → КОЛЛАПС (для наблюдателя на S=1)
  S=3: смотрит вниз на S=1: видит 9 × W(2) ≈ 6 тиков → НАРАСТАНИЕ

Наблюдатель S=3 не ошибается — он правильно видит
что происходит с его разрешения.
Наблюдатель S=1 тоже прав — на его уровне коллапс реален.
```

Варпинг W(S) описывает насколько сильно их реальности расходятся:
```
W(2) = 0.641: наблюдатель S=3 получает только 64% информации уровня S=1
```

## ЧАСТЬ VIII. SIFS-SCORE v6.1

Все расчёты в ln-пространстве.
Входные данные нормированы через размер тика **НА СВОЁМ УРОВНЕ**.

### Компоненты

**W1 — Объём/сила реакции**
```
W1 = ln(V_current / V_median) × φ⁻¹
φ⁻¹ ≈ 0.618
V_median — медиана за последние 89 наблюдений (Fib)
```

**W2 — Ускорение (с ограничением)**
```
W2 = clamp(ΔV/Δt × Ψ₂, −Ψ₁, +Ψ₁)
Ψ₂ ≈ 1.432, Ψ₁ ≈ 2.865
Ограничение ±Ψ₁ предотвращает выбросы на малых ТФ
```

**W3 — Накопленный взвешенный импульс**
```
W3 = Σ(weight_i × sign(tick_i))  за текущий цикл НА СВОЁМ УРОВНЕ
weight_i ∈ {0.5, 1.0, 1.5} по объёму тика
Обнуляется при коллапсе, переносится ×W(1) на S+1
```

**W4 — Отклонение от EMA в единицах BB**
```
W4 = (X_current − X_ema) / BB_width
Нормировка на BB_width делает W4 масштабно инвариантным
```

**W5 — Fractal Efficiency Ratio**
```
W5 = FER × sign(последнего тика)
FER = |X_end − X_start| / Σ|шагов| за окно 89
FER→1: направленное движение
FER→0: хаотичное движение
```

**W6 — Fractal Dimension**
```
W6 = (1.5 − D) × 0.5
D — размерность Хигучи, окно 89 (Fib)
D≈1.0: трендовое → W6 = +0.25
D≈1.5: случайное → W6 = 0.00
D≈2.0: хаотичное → W6 = −0.25
```

**W7 — Квантоподобный шум**
```
amplitude = 1/(π × Ψ₁ × GM) ≈ 0.0352  [выводится из констант]
W7 = ξ ~ N(0, amplitude)
В Contraction: amplitude × Ψ₂ ≈ 0.0504
```

**W8 — Multi-scale подтверждение (ИСПРАВЛЕНО В v6.1)**

```
W8 = Score(S-1) × W(1) × 0.50     [взгляд вниз: видим сжато]
   + Score(S)   × 0.35             [свой уровень: без искажения]
   + Score(S+1) / W(1) × 0.15      [взгляд вверх: декомпрессия]

где:
  W(1) = exp(-2k) ≈ 0.800  — компрессия при взгляде вниз
  1/W(1) ≈ 1.250           — декомпрессия при взгляде вверх

Логика:
  Когда смотрим ВНИЗ (на S-1):
    видим детали сжато через W(1), поэтому Score × W(1)
  Когда смотрим ВВЕРХ (на S+1):
    нужно восстановить масштаб, поэтому Score / W(1)
```

**Приоритет микро (с учётом компрессии):**
```
Если |Score(S-1) × W(1)| > 2×GM → W8 = Score(S-1) × W(1)

Это означает: даже сжатый Score микро-уровня
если превышает 2×GM=4.76 — переопределяет всё.

Пример:
  Score(S-1) = +6.0  →  после компрессии: 6.0 × 0.8 = +4.8
  4.8 > 2×GM=4.76  →  микро-приоритет активирован
```

### Финальный Score

```
SIFS_raw = W1 + W2 + W3 + W4 + W5 + W6 + W7 + W8

Contraction (FER<0.3, BB сжат, D>1.4):
  SIFS_final = SIFS_raw / (1 + |SIFS_raw| × π/Ψ₁)
  penalty_coefficient = π/Ψ₁ ≈ 1.097

Transition (FER 0.3–0.6):
  SIFS_final = SIFS_raw × 0.75
  + bias π/(n×100) на порогах 3/5/9

Expansion (FER>0.6, BB расширяется, D<1.3):
  SIFS_final = SIFS_raw
```

### Интерпретация

```
|Score| < 1.0    шум / нейтраль
|Score| ≥ GM     значимый сигнал (≥ 2.380)
|Score| ≥ π      сильный сигнал (≥ 3.142)
|Score| ≥ Ψ₁     сигнал смены уровня S (≥ 2.865)
```

## ЧАСТЬ IX. BIAS НАБЛЮДАТЕЛЯ

На порогах 3/5/9 наблюдатель вносит систематическое смещение:

```
Bias(n) = π / (n × 100)

Bias(3) = π/300  ≈ 0.01047
Bias(5) = π/500  ≈ 0.00628  ← максимальная неопределённость
Bias(9) = π/900  ≈ 0.00349
```

На уровне 5 тиков: система не может знать — истина это или эхо,
не изменив результат наблюдения. Это аналог принципа неопределённости.

## ЧАСТЬ X. АЛГОРИТМ РАБОТЫ

```
ШАГ 0. ПАРАМЕТРЫ
  → Определить 1 тик через границы (BB/EMA/аналог)
  → Определить S=0 (последний подтверждённый экстремум)
  → Определить уровень S (масштаб наблюдения)
  → Проверить: если накоплено >9 тиков НА СВОЁМ УРОВНЕ → пересчитать S=0

ШАГ 1. СЧЁТ ТИКОВ НА СВОЁМ УРОВНЕ
  → Подсчитать взвешенные тики от S=0 НА СВОЁМ УРОВНЕ S
  → Классифицировать: Шум / Истина / Эхо / Коллапс
  → Проверить BB-состояние и RSI-зону

ШАГ 2. КОМПОНЕНТЫ SCORE
  → Вычислить W1–W7 в ln-пространстве

ШАГ 3. MULTI-SCALE (W8) С КОМПРЕССИЕЙ
  → Запросить Score с S-1 и S+1
  → Применить компрессию:
     Score(S-1) × W(1)  — взгляд вниз
     Score(S+1) / W(1)  — взгляд вверх
  → Вычислить W8 с весами 0.50 / 0.35 / 0.15

ШАГ 4. ФАЗА
  → Определить Contraction / Transition / Expansion
  → Применить соответствующий penalty или bias

ШАГ 5. ИТОГОВЫЙ SCORE
  → Вычислить SIFS_final
  → Интерпретировать: шум / значимый / сильный / смена уровня

ШАГ 6. ПРОГНОЗ
  → Построить уровни Fib×π от S=0 НА СВОЁМ УРОВНЕ
  → Указать ближайшие точки реакции с вероятностями

ШАГ 7. КОЛЛАПС (если 9 тиков НА СВОЁМ УРОВНЕ)
  → W3 × W(1) → передать на S+1
  → S=0 = точка коллапса
  → BB начинает сжиматься → подтверждение
  → Начать новый цикл
```
