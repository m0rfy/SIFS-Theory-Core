# Руководство по миграции терминологии SIFS Theory

Этот документ содержит инструкции по переходу от старой терминологии к новой, а также план отката (rollback) в случае проблем.

## Введение

Миграция терминологии необходима для обеспечения академической строгости и предотвращения концептуальной путаницы. Этот гайд поможет безопасно обновить все документы.

---

## Таблица замен

### Основные замены

| Старый термин | Новый термин | Контекст | Пример |
|--------------|-------------|----------|--------|
| "черная дыра" (микро) | "effective horizon" / "horizon-like trapped region" | Элементарные частицы, микроуровень | "Протон — это effective horizon-like trapped region" |
| "подтверждено данными" | "согласуется с данными" | Наблюдательные данные | "Наблюдения согласуются с предсказаниями" |
| "доказано" | "предсказано теорией" | Предсказания | "SIFS предсказывает..." |
| "экспериментальное подтверждение" | "наблюдательное согласие" | Эксперименты | "Наблюдательное согласие с теорией" |

### Разрешенные контексты для "черная дыра"

✅ **Разрешено использовать "черная дыра" для:**
- Астрофизических объектов (M87*, Sgr A*)
- Макроскопических ЧД (M > M_☉)
- Теоретических обсуждений (информационный парадокс, кривая Пейджа)
- Исторических ссылок на работы других авторов

❌ **Запрещено использовать "черная дыра" для:**
- Элементарных частиц (протон, электрон)
- Микроскопических масштабов
- Прямого описания структуры частиц

## Процесс миграции

### Шаг 1: Подготовка

1. **Создать резервную копию:**
   ```bash
   git branch backup-before-terminology-migration
   git checkout -b terminology-migration
   ```

2. **Запустить скрипт проверки:**
   ```powershell
   .\scripts\check-terminology.ps1 -Path docs/
   ```

3. **Просмотреть список проблем:**
   - Зафиксировать все найденные проблемы
   - Определить приоритеты исправления

### Шаг 2: Систематическое исправление

#### Группа 1: Теоретические документы
- `docs/theory/*.md`
- Начать с `overview.md` как основного документа

#### Группа 2: Документы защиты
- `docs/defense/*.md`
- Особое внимание к `thermodynamics-defense.md`

#### Группа 3: Протоколы и расчеты
- `docs/protocol/*.md`
- `docs/calculations/*.md`

#### Группа 4: Предсказания
- `docs/predictions/*.md`
- Заменить "подтверждено" на "согласуется"

### Шаг 3: Проверка после изменений

1. **Запустить скрипт проверки снова:**
   ```powershell
   .\scripts\check-terminology.ps1 -Path docs/
   ```

2. **Проверить ссылки:**
   ```powershell
   .\scripts\validate-links.ps1 -Path docs/
   ```

3. **Проверить формулы:**
   ```powershell
   .\scripts\validate-formulas.ps1 -Path docs/
   ```

### Шаг 4: Коммит изменений

```bash
git add docs/
git commit -m "Миграция терминологии: замена 'черная дыра' → 'effective horizon', 'подтверждено' → 'согласуется'"
```

## Примеры правильных и неправильных замен

### Пример 1: Протон

#### ❌ Неправильно

> "Протон — это черная дыра с массой 10¹⁴ г в bulk. Это подтверждено расчётами."

#### ✅ Правильно

> "Протон моделируется как effective horizon-like trapped region в 5D-bulk с истинной массой M_bulk ≈ 10¹⁴ г. Расчёты показывают, что наблюдаемая масса m_p ≈ 10⁻²⁴ г согласуется с RS-warping фактором exp(-2k|S|) при |S| ≈ 11.2."

### Пример 2: DESI 2025

#### ❌ Неправильно

> "DESI 2025 подтвердил предсказания SIFS об эволюционирующей тёмной энергии."

#### ✅ Правильно

> "Наблюдения DESI 2025 согласуются с предсказаниями SIFS об эволюционирующей тёмной энергии. Измеренное значение w₀ = -0.827 ± 0.063 отклоняется от ΛCDM на >4σ, что согласуется с предсказанным дрейфом глобальной масштабной координаты S_global."

### Пример 3: Константы связи

#### ❌ Неправильно

> "SIFS доказала, что все константы выводятся из одной геометрической аксиомы."

#### ✅ Правильно

> "SIFS предсказывает, что все фундаментальные константы связи могут быть выведены из одной геометрической аксиомы (5D метрика) через масштабную координату S. Численные расчёты показывают согласие с экспериментальными значениями в пределах ошибок."

## План отката (Rollback Plan)

Если после миграции обнаружены проблемы:

### Вариант 1: Частичный откат

```bash
# Откатить конкретный файл
git checkout HEAD~1 -- docs/theory/overview.md
```

### Вариант 2: Полный откат

```bash
# Вернуться к ветке backup
git checkout backup-before-terminology-migration
git branch -D terminology-migration
```

### Вариант 3: Исправление ошибок

Если ошибки обнаружены после коммита:

1. Создать новую ветку для исправлений
2. Исправить проблемы
3. Создать новый коммит с исправлениями

## Часто задаваемые вопросы

### Q: Можно ли использовать "черная дыра" в кавычках?

A: Да, если это цитата из другого источника или историческая ссылка. Но в собственном тексте используйте "effective horizon".

### Q: Что делать с таблицами сравнения?

A: В таблицах сравнения можно оставить "черная дыра" если сравниваете с астрофизическими объектами. Для микроуровня используйте "effective horizon".

### Q: Как обрабатывать ссылки на другие работы?

A: В ссылках на работы других авторов можно оставить оригинальную терминологию, но добавить примечание о терминологии SIFS.
