# Инструкции для AI агента в Cursor IDE

**Важно:** Этот проект использует **только встроенных агентов Cursor IDE**. Мы не используем внешние агенты (Gemini, OpenCode, Claude Code и т.д.). Все правила и инструкции оптимизированы для работы с агентами Cursor IDE.

Этот файл содержит общие инструкции для AI агента Cursor IDE при работе с проектом SIFS-Theory-Core.

## Основные принципы работы

### 1. Выполнение задач

- **Один терминал = одна задача** - не смешивать несколько задач в одном терминале
- **Последовательное выполнение** - дожидаться завершения предыдущей команды
- **Проверка статуса** - проверять результат перед следующим шагом
- **Обработка ошибок** - обрабатывать ошибки немедленно, не продолжать при ошибках

### 2. Работа с кодом

- **TypeScript** - использовать строгую типизацию, избегать `any`
- **React 18+** - функциональные компоненты с хуками
- **Tailwind CSS** - только utility классы, без inline стилей
- **shadcn/ui** - использовать существующие компоненты из `src/app/components/ui/`

### 3. Оптимизация контекста

- **Приоритизация файлов:**
  - Критически важные: `package.json`, `README.md`, `vite.config.ts`, `src/main.tsx`
  - Важные: компоненты в `src/app/components/ui/`, конфигурационные файлы
  - Справочные: документация в `docs/`, скрипты в `scripts/`
- **Минимизация запросов** - группировать связанные чтения, использовать поиск
- **Кэширование** - запоминать структуру проекта, технологии, паттерны

## Режимы работы агентов Cursor IDE

Проект использует различные режимы работы агентов Cursor IDE в зависимости от ситуации:

### Agent Mode (Режим агента)

- **Назначение:** Выполнение сложных задач, рефакторинг, редактирование нескольких файлов
- **Возможности:** Автономное исследование кодовой базы, внесение изменений, запуск команд в терминале
- **Когда использовать:** Для задач требующих активного изменения кода и структуры проекта

### Ask Mode (Режим вопросов)

- **Назначение:** Обучение, планирование, анализ кода
- **Возможности:** Поиск по кодовой базе, предоставление ответов без внесения изменений
- **Когда использовать:** Для изучения проекта, планирования изменений, получения консультаций

### Custom Modes (Пользовательские режимы)

- **Назначение:** Специализированные режимы с пользовательскими настройками
- **Возможности:** Настройка инструментов и инструкций под конкретные рабочие процессы
- **Когда использовать:** Для специфических задач требующих особой конфигурации

**Важно:** Проект **не использует** Background Agent и теневые запуски. Все задачи выполняются в видимом режиме, чтобы пользователь мог видеть что происходит в реальном времени.

## Правила Cursor

Проект использует правила Cursor IDE для оптимизации работы. В зависимости от ситуации применяются разные подробные правила и инструкции:

- **task-execution** - управление выполнением задач (применяется всегда)
- **docker-usage** - работа с Docker (для Docker файлов)
- **project-context** - контекст проекта (применяется всегда)
- **frontend-agent** - разработка frontend (для frontend файлов)
- **cursor-optimization** - оптимизация работы (применяется всегда)
- **mcp** - работа с MCP серверами (для MCP файлов)
- **cli-background** - CLI и Background Agent (для автоматизации)

## Навыки (Skills)

Проект использует skills для предоставления специализированных инструкций по выполнению конкретных задач:

- **code-analysis** - систематический анализ кода
- **component-creation** - создание React компонентов
- **debugging** - поиск и исправление ошибок
- **documentation** - работа с документацией
- **refactoring** - улучшение структуры кода
- **testing** - создание и выполнение тестов

Skills находятся напрямую в `.cursor/` как `.cursor/<skill-name>/SKILL.md`. Каждый skill соответствует [спецификации Agent Skills](https://agentskills.io/specification).

## Команды Cursor

Доступные slash-команды:

- `/task-queue` - управление очередью задач
- `/docker-check` - проверка Docker контейнеров
- `/optimize-context` - оптимизация контекста
- `/frontend-dev` - разработка frontend компонентов

## MCP серверы

Проект использует следующие MCP серверы:

- **OpenAI Image Generation** - генерация изображений
- **GitHub MCP** - работа с GitHub
- **Shadcn MCP** - поиск компонентов shadcn/ui
- **Browser MCP** - тестирование веб-приложений

## Важные замечания

1. **Кодировка** - всегда использовать UTF-8
2. **Обработка ошибок** - при ошибках в веб-приложении делать скриншот, анализировать консоль
3. **Последовательность** - выполнять задачи по очереди, не переходить к следующей пока не завершена предыдущая
4. **Документация** - проект содержит обширную научную документацию на русском языке
